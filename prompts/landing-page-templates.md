# Landing Page Prompt Templates

Based on MemberPress landing page structures. Use these templates to generate high-converting landing pages with AI.

---

## Template 1: Feature Overview Page (Grid Style)

**Use for:** Main features/capabilities page showcasing multiple features at once
**Example:** https://memberpress.com/features/

### Prompt Template

```
Write a feature overview landing page in markdown for: [PRODUCT NAME]

Product Description: [BRIEF DESCRIPTION]
Target Audience: [WHO]
Main Value Proposition: [CORE BENEFIT]

Structure:

# [Compelling Headline - Main Value Proposition]

[Subheadline - who it's for and what problem it solves]

[1-2 sentence introduction paragraph]

## Hero Section

**Primary CTA:** [Get Started / Try Free / See Demo]

[Brief paragraph explaining the core promise - conversational, benefit-focused]

## Major Features (6 core capabilities)

For each feature:
### [Feature Name - Benefit-Driven Headline]

[2-3 sentences explaining what it does and why it matters]

**Key benefits:**
- [Benefit 1]
- [Benefit 2]
- [Benefit 3]

[Suggested image description: screenshot or icon representing this feature]

---

## Additional Capabilities (grid of 4-8 smaller features)

Present as short feature blocks:

### [Capability Name]
[1 sentence description]

### [Capability Name]
[1 sentence description]

[Continue for each capability...]

---

## Integration & Automation

### [Integration Category 1]
[Brief description of integration capabilities]

### [Integration Category 2]
[Brief description of automation features]

---

## Social Proof

> "[Customer testimonial highlighting specific benefit]"
> **- [Name, Title, Company]**

> "[Another testimonial showing different use case]"
> **- [Name, Title, Company]**

**Trusted by [NUMBER]+ businesses** [or similar social proof stat]

**Ratings:** [mention review sites and scores if available]

---

## Final CTA Section

# [Reinforcing Headline]

[1-2 sentences creating urgency or reinforcing value]

**[Primary CTA Button]**

[Secondary CTA or guarantee statement]

---

Writing Style:
- Tone: Conversational yet professional
- Focus: Benefits over features (translate "what" into "why it matters")
- Language: Use "you/your" to speak directly to reader
- Pattern: Problem → Solution → Outcome
- CTAs: Action-oriented verbs ("Get," "Start," "Build," "Create")
- Simplicity: Emphasize ease of use ("No tech skills required," "Setup in minutes")

Visual Cues to Include:
- [Screenshot: feature interface]
- [Icon: for each major feature]
- [Badge: review site ratings]
- [Photo: testimonial sources]
```

## Template 2: Single Product Feature Page (Deep Dive)

**Use for:** Dedicated landing page for one specific product feature or add-on
**Example:** https://memberpress.com/features/coachkit-coaching-platform-wordpress/

### Prompt Template

```
Write a product feature landing page in markdown for: [FEATURE/PRODUCT NAME]

Main Product: [PARENT PRODUCT]
Feature Description: [WHAT IT DOES]
Target Audience: [WHO NEEDS THIS]
Key Differentiator: [WHAT MAKES IT UNIQUE]

Structure:

# [Aspirational Headline - The Outcome This Enables]

[Metaphorical or emotional subheadline that resonates with target audience's ambitions]

## Introduction

[2-3 paragraphs using aspirational language to position the product. Use metaphors and emotional appeals while maintaining professionalism. Frame the product as enabling the reader's goals.]

**[Primary CTA: Get Started / Learn More]**

---

## Feature Deep Dive (6 major sections)

Use alternating layout pattern (image left, text right, then reverse):

### [Feature 1 - Benefit-Focused Headline]

[Opening sentence with emotional hook or outcome focus]

[2-3 sentences explaining the capability in detail, connecting features to business outcomes]

**What this means for you:**
- [Specific benefit 1]
- [Specific benefit 2]  
- [Specific benefit 3]

[Visual description: Screenshot showing this feature in action]

---

### [Feature 2 - Benefit-Focused Headline]

[Repeat pattern, varying the emotional angle and specific benefits]

[Continue for all 6 features...]

---

## Why This Matters

[Section explaining the bigger picture - how this fits into their business strategy]

**The result:**
- [Business outcome 1]
- [Business outcome 2]
- [Business outcome 3]

---

## Frequently Asked Questions

### [Question 1 - addressing common objection or technical detail]
[Clear, straightforward answer]

### [Question 2 - about implementation or compatibility]
[Answer with technical specifics when needed]

### [Question 3 - about pricing or limitations]
[Honest answer that removes friction]

### [Question 4 - about support or getting started]
[Reassuring answer with next steps]

---

## Social Proof

**Rated [X.X]/5** across industry-leading review platforms:
- Capterra: [Rating]
- G2: [Rating]
- Trustpilot: [Rating]

**Awards:**
- [Award 1]
- [Award 2]

---

## Ready to Get Started?

# [Reinforcing Headline - Calls Back to Opening Theme]

[Final paragraph creating urgency or reinforcing transformation]

**[Primary CTA Button]**

[Trust statement: money-back guarantee, free trial, etc.]

---

Writing Style:
- Tone: Aspirational yet credible, balancing emotion with professionalism
- Metaphors: Use throughout (e.g., "climb," "summit," "journey," "foundation")
- Emotional Language: Focus on transformation and outcomes
- Technical Balance: Keep technical details in FAQ, benefits in main sections
- Progressive Build: Start simple, build to advanced capabilities
- "You" Language: Make it personal and direct
- Outcome Focus: Every feature connects to a business result

Visual Cues to Include:
- [Hero image: aspirational/metaphorical representation]
- [Feature screenshots: alternating sides for visual rhythm]
- [Icons: for FAQ sections]
- [Badges: review site ratings and awards]
- [Gradient backgrounds: create visual separation between sections]

Unique Elements for This Style:
- Use metaphorical headlines (e.g., "No Caps, No Limits" vs. "Unlimited Users")
- Frame features as enablers of ambition, not just tools
- Emphasize scalability and growth potential
- Include more emotional/aspirational language
- Deeper FAQ section addressing objections
```

## Template 3: Comparison Landing Page (Alternative Style)

**Use for:** Pages comparing your product to competitors or showing "why choose us"

### Prompt Template

```
Write a comparison landing page in markdown for: [PRODUCT]

Comparing to: [COMPETITOR / Alternative Approach]
Key Advantages: [3-5 main differentiators]

Structure:

# [Headline Positioning Your Advantage]

[Subheadline showing who this comparison is for]

## The Challenge

[Describe the common pain point or frustration with current solutions]

**Common problems with [alternative approach]:**
- [Pain point 1]
- [Pain point 2]
- [Pain point 3]

## Why [Your Product] is Different

### [Differentiator 1]
**Them:** [What competitors do]  
**Us:** [What you do better]

[2-3 sentences explaining the advantage]

### [Differentiator 2]
**Them:** [What competitors do]  
**Us:** [What you do better]

[2-3 sentences explaining the advantage]

[Continue for each differentiator...]

## Side-by-Side Comparison

| Feature | [Competitor/Alternative] | [Your Product] |
|---------|-------------------------|----------------|
| [Feature 1] | [Their approach] | ✅ [Your advantage] |
| [Feature 2] | [Their approach] | ✅ [Your advantage] |
| [Feature 3] | Limited/Paid extra | ✅ Included |
| [Feature 4] | [Their approach] | ✅ [Your advantage] |

## Real Results

> "[Testimonial from someone who switched]"
> **- [Name, previous solution they used]**

## Make the Switch

[Section explaining how easy it is to migrate/get started]

**[Primary CTA]**

---

Writing Style:
- Honest and factual (avoid trash-talking competitors)
- Focus on your strengths, not their weaknesses
- Use checkmarks and visual indicators
- Include migration support/resources
- Emphasize ease of switching
```

## Usage Instructions

1. **Choose the template** that matches your page goal
2. **Fill in the bracketed sections** with your specific content
3. **Paste into Claude/ChatGPT** to generate the content
4. **Refine the output** by asking for adjustments
5. **Save as markdown file**
6. **Publish with this tool:**
   ```bash
   python3.11 publish.py your-landing-page.md --status draft --post-type pages
   ```

## Pro Tips

- **Template 1** works best for main feature overview pages
- **Template 2** works best for deep-dive product pages or add-on features
- **Template 3** works best for competitive positioning
- Mix and match sections from different templates as needed
- Always include social proof and multiple CTAs
- Use aspirational language for Template 2, practical language for Template 1
- Test draft versions before publishing live

## Example Combinations

**For a SaaS product launch:**
1. Use Template 1 for main features page
2. Use Template 2 for each major feature (create 3-5 pages)
3. Use Template 3 for "vs. competitors" page

**For a WordPress plugin:**
1. Use Template 1 for main plugin page
2. Use Template 2 for premium/pro features
3. Use Template 3 for "vs. other plugins" comparison
